# Assessment Guide - Project Management

## Overview
Panduan untuk memahami assessment waktu pengerjaan setiap task dalam project.

## Assessment Methodology

### Definisi Assessment
Assessment adalah estimasi waktu yang diperlukan untuk menyelesaikan setiap task/subtask dalam project, berdasarkan:
- Kompleksitas task
- Skill level developer
- Dependencies dengan task lain
- Historical data dari project serupa

### Time Unit
- Diukur dalam **hari** (1 hari = 8 jam kerja)
- 1 minggu = 5 hari kerja

### Assessment Levels

#### Level 1: Simple Tasks
- Waktu: 1 hari
- Contoh: Setup, documentation update, minor bug fixes
- Kompleksitas: Rendah

#### Level 2: Medium Tasks  
- Waktu: 2-3 hari
- Contoh: Component development, API integration, testing
- Kompleksitas: Sedang

#### Level 3: Complex Tasks
- Waktu: 4-6 hari
- Contoh: Full feature development, performance optimization
- Kompleksitas: Tinggi

#### Level 4: Very Complex Tasks
- Waktu: 7+ hari
- Contoh: System architecture, major refactoring
- Kompleksitas: Sangat Tinggi

## Project Status

### Status Types
1. **Planning** - Fase perencanaan & requirement gathering
2. **In Progress** - Sedang dikerjakan
3. **Review** - Menunggu review/approval
4. **Completed** - Sudah selesai
5. **On Hold** - Ditunda sementara

## PM Query Examples

Pertanyaan yang bisa diajukan kepada chat:

1. "Berapa estimasi waktu untuk Website Redesign project?"
2. "Apa saja tasks di Mobile App project?"
3. "Project mana yang sudah 60% selesai?"
4. "Berapa total hari untuk frontend development?"
5. "Apa status project-project saat ini?"
6. "Kapan Mobile App project selesai?"

## Notes
- Assessment bersifat estimasi, bisa berubah sesuai kondisi
- Buffer time untuk bug fix dan review sudah termasuk
- Dependencies antar task harus diperhatikan saat eksekusi
